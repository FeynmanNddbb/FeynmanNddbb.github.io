我的AI生产力工具栈配置思路

发表于2026-03-09|更新于2026-03-09|生产力
|总字数:594|阅读时长:1分钟|浏览量:

最近整理了一下自己的AI工具栈，发现不知不觉已经形成了比较清晰的分工。分享一下目前的配置思路和背后的考虑。

工具分工一览

工具 账号类型 主要用途 可用模型
Antigravity Google 学生账号 文档、任务、Skill 类 Claude、Gemini（Thinking / Lite 额度独立）
GitHub Copilot GitHub 学生账号 开发、工程、Coding 类 GPT、Claude
OpenClaw 国产大模型 Plan 套餐 娱乐、轻度使用 国内模型为主

各工具详细说明

📝 Antigravity — 文档与任务中枢

使用 Google 学生账号 接入，主要优势：

  • 支持 Claude 和 Gemini 双模型
  • Thinking 和 Lite 模型额度独立，可以根据任务复杂度灵活切换
  • 适合长文档处理、任务拆解、知识管理类工作

对于需要深度思考的内容，我会用 Thinking 模型；日常快速问答则用 Lite 节省额度。

💻 GitHub Copilot — 开发专属

学生账号福利，专注于代码场景：

  • 支持 GPT 和 Claude 模型
  • 代码补全、重构、Debug 一体化
  • 与 IDE 深度集成，开发体验流畅

Coding 类需求基本全部交给 Copilot，专业的事交给专业的工具。

🤖 OpenClaw — 娱乐与轻量使用

选择了国产大模型的 Plan 套餐，原因很实际：

反代国外模型容易封号，且我的用量需求并不大。与其冒险折腾，不如买个国内套餐支持国产，稳定省心。

主要用于：

  • 日常闲聊
  • 快速查资料
  • 轻度创意发散

🔮 Claude Code — 观望中

目前暂时未配置，原因如下：

  1. Skill 类需求已有 Antigravity 覆盖
  2. Coding 类需求已有 Copilot 覆盖
  3. 最重要的 —— 坐等降价

Token 内卷是大趋势，价格只会越来越便宜。现在不急着上车，等官方下场补贴 OpenClaw，长期利好电网和光模块 😄

等价格合适了再考虑接入，目前这套组合已经够用了。

总结

这套配置的核心思路是**”专业分工 + 风险分散”**：

  • 不同场景用最适合的工具
  • 学生账号薅羊毛（合规使用）
  • 国产工具兜底，避免封号风险
  • 对新工具保持观望，不盲目追新

如果你也在搭建自己的AI工具栈，希望这篇分享能给你一些参考。
