我的AI生产力工具全家桶

发表于2026-03-09|更新于2026-03-09|生产力工具
|总字数:1.3k|阅读时长:3分钟|浏览量:

作为一个重度依赖AI提升效率的用户，我的工具箱里常年保持着几位”得力助手”。它们各有专长，相互配合，覆盖了我工作和生活的不同场景。今天来分享一下这套AI生产力工具全家桶的配置思路和实际使用体验。

🎯 工具总览

工具 账号类型 核心用途 主打模型
Antigravity Google 学生账号 文档、任务、Skill 类 Claude、Gemini
GitHub Copilot GitHub 学生账号 开发、工程、Coding 类 GPT、Claude
OpenClaw 国产大模型 Plan 套餐 日常娱乐、轻度使用 国产模型 + 搜索

一、Antigravity：文档与任务的智能中枢

账号： Google 学生账号

Antigravity 是我处理非代码类任务的首选工具。通过 Google 学生账号，可以解锁 Claude 和 Gemini 双模型支持，而且 Thinking 模型和 Lite 模型的额度是独立的，这一点非常实用。

适用场景

  • 📄 文档撰写：报告、总结、邮件、文案
  • ✅ 任务管理：待办拆解、计划制定、进度追踪
  • 🔧 Skill 开发：构建自动化工作流、提示词工程
  • 💡 创意发散：头脑风暴、方案对比、决策辅助

使用心得

学生账号的额度对于日常办公来说完全够用。Thinking 模型适合需要深度推理的复杂任务，Lite 模型则用于快速问答和简单处理，两者搭配效率翻倍。

二、GitHub Copilot：开发者的代码搭档

账号： GitHub 学生账号

Copilot 是编程场景下的绝对主力。同样通过学生账号获取，支持 GPT 和 Claude 模型，在代码补全、重构建议、Bug 排查等方面表现出色。

适用场景

  • 💻 日常编码：代码补全、函数生成、注释编写
  • 🏗️ 工程开发：项目架构设计、技术选型建议
  • 🔍 Debug 辅助：错误分析、日志解读、修复方案
  • 📚 学习提升：源码解读、算法讲解、最佳实践

使用心得

Copilot 的上下文理解能力很强，尤其是在大型项目中，能够基于已有代码风格生成一致的代码。学生账号的免费额度对于个人开发和学习来说绰绰有余。

三、OpenClaw：国产替代与日常娱乐

套餐： 国产大模型 Plan 套餐（Searching 功能）

OpenClaw 在我的工具箱里扮演补充角色。选择它的原因有三：

  1. 反代国外模型容易封号 —— 稳定性考虑
  2. 用量需求不大 —— 轻度使用场景
  3. 支持国产 —— 买个国内套餐，支持下本土AI发展

适用场景

  • 🎮 日常娱乐：闲聊、趣味问答、轻松互动
  • 🔍 联网搜索：需要实时信息的查询场景
  • 📱 移动端轻量使用：快速获取答案

使用心得

国产模型的进步有目共睹，对于不需要深度推理的日常场景完全够用。而且 OpenClaw 的搜索功能在查资料时很方便，是一个不错的补充工具。

四、Claude Code：暂时观望，等待时机

状态： 暂未配置，持续观望中

按理说，有了 Antigravity 处理文档、Copilot 处理代码，Claude Code 似乎是个完美的补充。但我目前的策略是：先等等。

观望理由

因素 分析
Token 价格 内卷是大趋势，价格正在快速下降
官方补贴 OpenClaw 官方下场补贴，长期利好用户
需求覆盖 现有工具已能满足 90% 的场景
电网 & 光模块 基础设施完善后，成本会进一步降低

未来计划

等 Token 价格降到心理预期，或者官方推出更优惠的套餐时，再考虑接入 Claude Code。毕竟等等党永远不亏，而且现在的组合已经足够高效。

五、工具选型思路总结

这套组合的核心原则是：场景细分 + 成本优化 + 风险分散。

几个关键决策点

  1. 学生账号优先 —— 教育优惠是真香，额度够用还免费
  2. 模型互补 —— 不同平台的主打模型不同，互为备份
  3. 国产支持 —— 轻度场景用国产，既省钱又支持本土AI
  4. 动态调整 —— 市场变化快，保持灵活，该等等就等等

写在最后

AI 工具的选择没有标准答案，关键是找到适合自己 workflow 的组合。我的这套配置是基于当前的使用习惯、成本考量和风险分散策略形成的，随着市场变化和个人需求演进，也会持续调整。

如果你也在搭建自己的 AI 工具箱，希望这篇分享能给你一些参考。有什么想法或问题，欢迎在评论区留言交流！

💡 小提示：学生账号的申请门槛并不高，如果你符合条件，强烈建议申请一下。教育优惠的额度对于个人使用来说非常 generous，能省则省嘛！
